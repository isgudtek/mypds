pdscheck is a static page. It runs entirely in your browser. There is no server, no analytics, no logging — just fetch() calls to your PDS, multiple public relays, and plc.directory.
Some probes require CORS to be enabled on your PDS. If a probe fails with a CORS error, expand the row and run the printed curl command from your terminal — that's the source of truth.
Probe and relay-fanout design influenced by debug.hose.cam by fig & bailey (MIT/Apache-2.0). Built for self-hosters; probe registry is a single array — add your own.
